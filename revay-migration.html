üîß Step-by-Step Migration Procedure

Phase 1 ‚Äî Pre-Migration (on old Cloudflare container)

1
Push workspace to GitHub (backup)
cd /root/.openclaw/workspace
git add -A
git commit -m "pre-migration backup $(date -u +%Y-%m-%dT%H:%M:%SZ)"
git push origin master
This creates a cloud backup of all memory files, scripts, and Salesforce tokens.
2
Export full gateway config
openclaw gateway call config.get --json > /tmp/openclaw-config-export.json
cat /tmp/openclaw-config-export.json
Save this output ‚Äî it contains all settings needed to restore runtime configuration.
3
Export cron job definitions
openclaw cron list --json > /tmp/cron-backup.json
cat /tmp/cron-backup.json
Cron jobs (10 active, including one time-sensitive one-shot alert) are stored in gateway state. Save this file.
4
Save auth-profiles.json
cat /root/.openclaw/agents/main/agent/auth-profiles.json
Save this output ‚Äî contains Anthropic + OpenAI API keys not stored in openclaw.json.
5
Tar the full .openclaw directory
tar -czf /tmp/openclaw-full-backup.tar.gz /root/.openclaw/
ls -lh /tmp/openclaw-full-backup.tar.gz
Transfer this tar file to a secure location (Azure Blob Storage or Evan's local machine).

Phase 2 ‚Äî Azure Container Setup (on new container)

1
Install Node.js v22 LTS
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt-get install -y nodejs
node --version  # must show v22.x
2
Install OpenClaw (same version)
npm install -g openclaw@2026.2.21-2
Or restore from the tar backup (preserves exact binary version).
3
Restore .openclaw directory from backup
tar -xzf openclaw-full-backup.tar.gz -C /
This restores config, workspace, session history, cron state, and auth profiles in one step.
4
Set git credentials (for workspace GitHub sync)
echo "https://erice320:[GITHUB_TOKEN]@github.com" > ~/.git-credentials
git config --global credential.helper store
git config --global user.email "revayr23@gmail.com"
git config --global user.name "Revay"
GitHub token is in workspace TOOLS.md.
5
Install workspace npm dependencies
cd /root/.openclaw/workspace && npm install
6
Start OpenClaw gateway
openclaw gateway start
openclaw gateway status
7
Rebuild memory search index
openclaw memory index
Must be run after workspace is restored. Indexes ~60 files / ~329 chunks for semantic memory search.
8
Configure process supervisor (keep-alive)
Using systemd: 
[Unit]
Description=Revay OpenClaw Gateway
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/openclaw gateway start --foreground
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
systemctl enable revay && systemctl start revay
